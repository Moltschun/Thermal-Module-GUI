# Thermal-Module-GUI

**Профессиональный графический интерфейс для управления промышленными тепловизионными модулями.**

## О проекте

**Thermal-Module-GUI** — это высокопроизводительное приложение для захвата, визуализации и записи данных с тепловизионных камер высокого разрешения. 

Проект разработан для решения проблем высокой задержки (latency) и мерцания изображения (flickering) при работе с потоковым видео 1080p в среде Python/Qt. Архитектура приложения построена на модульном принципе, позволяя легко переключаться между различными физическими интерфейсами (USB3 Vision, Gigabit Ethernet / RTSP).

---

## Поддерживаемое оборудование

Проект включает драйверы для двух типов промышленных модулей:

| Производитель | Модель | Интерфейс | Протокол | Примечание |
|---|---|---|---|---|
| **Pergam / Dali** | Axion Uncooled HD 1920 | Ethernet (RJ45) | RTSP / ONVIF | Поддержка RTSP-ссылок DALI Web API |
| **Teledyne FLIR** | Blackfly S (BFS-U3) | USB 3.1 | Spinnaker SDK | Используется проприетарный драйвер PySpin |

> **Особенность Pergam:** Реализован обход проблемы авторизации ONVIF (рассинхронизация времени) и поддержка нестандартных портов (8000/554).

## Ключевые возможности

### 1. Zero-Copy Rendering (Без мерцания)
Вместо стандартной медленной конвертации `Base64` -> `QML`, используется кастомный `QQuickImageProvider`. Данные из буфера OpenCV/Spinnaker передаются в видеопамять по ссылке, минуя лишнее копирование. Это обеспечивает **стабильные 25-60 FPS** без нагрузки на сборщик мусора (GC).

### 2. Запись Visual RAW без потерь
Реализован асинхронный модуль записи `FrameRecorder`. 
* **Буферизация:** Кадры мгновенно сохраняются в оперативную память.
* **Фоновая запись:** Отдельный поток сбрасывает данные на диск в формате **TIFF** (Visual Raw), исключая "фризы" интерфейса во время съемки.

### 3. Low-Latency Network (для Pergam)
Для работы с RTSP используются оптимизированные флаги FFMPEG:
* `fflags=nobuffer` — отключение внутренней буферизации.
* `rtsp_transport=udp` — форсирование UDP для снижения задержки.

### 4. Цифровая обработка сигналов (DSP)
* **Pergam:** Программный Gain (усиление яркости) и Gamma-коррекция в реальном времени.
* **FLIR:** Управление аппаратной выдержкой и усилением сенсора.
